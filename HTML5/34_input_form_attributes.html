HTML Input Form Attributes

The input form attribute specifies the form the <input> element belongs to.
The value of this attribute must be equal to the id attribute of the <form> element it belongs to.

Example Code Output
An input field located outside of the HTML form (but still a part of the form):
<form action="save.php" id="form1">
     <label for="name">Name:</label>
     <input type="text" id="name" name="name">
     <input type="submit" value="Submit">
</form>
<label for="mobile">Mobile:</label>
<input type="text" id="mobile" name="mobile" form="form1">




HTML Input formaction Attribute

The input formaction attribute specifies the URL of the file that will process the input when the form is submitted.
This attribute overrides the action attribute of the <form> element.
The formaction attribute works with the following input types: submit and image.

Example Code Output
An HTML form with two submit buttons, with different actions:
<form action="save.php">
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit">
     <input type="submit" formaction="save2.php" value="Submit as Admin">
</form>




HTML Input formenctype Attribute

The input formenctype attribute specifies how the form-data should be encoded when submitted (only for forms with method="post").
This attribute overrides the action attribute of the <form> element.
The formenctype attribute works with the following input types: submit and image.

Example Code Output
A form with two submit buttons. The first sends the form-data with default encoding, the second sends the form-data encoded as "multipart/form-data":
<form action="save.php" method="POST">
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit">
     <input type="submit" formenctype="multipart/form-data" value="Submit as Multipart/FormData">
</form>



HTML Input formmethod Attribute

The input formmethod attribute defines the HTTP method for sending form-data to the action URL.
This attribute overrides the action attribute of the <form> element.
The formmethod attribute works with the following input types: submit and image.
The form-data can be sent as URL variables (method="get") or as an HTTP post transaction (method="post").

Example Code Output
A form with two submit buttons. The first sends the form-data with method="get". The second sends the form-data with method="post"
<form action="save.php" method="GET">
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit using GET">
     <input type="submit" formmethod="post" value="Submit using POST">
</form>




HTML Input formtarget Attribute

The input formtargetattribute specifies a name or a keyword that indicates where to display the response that is received after submitting the form.
This attribute overrides the action attribute of the <form> element.
The formtarget attribute works with the following input types: submit and image.

Example Code Output
A form with two submit buttons, with different target windows
<form action="save.php" method="POST">
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit">
     <input type="submit" formtarget="_blank" value="Submit to a New Window/Tab">
</form>




HTML Input formnovalidate Attribute

The input formtarget attribute specifies that an element should not be validated when submitted.
This attribute overrides the action attribute of the <form> element.
The formtarget attribute works with the following input types: submit.

Example Code Output
A form with two submit buttons (with and without validation):
<form action="save.php" method="POST">
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit">
     <input type="submit" formnovalidate="formnovalidate" value="Submit without Validation">
</form>




HTML Input formnovalidate Attribute

The input novalidate attribute is a <form> attribute.
When present, novalidate specifies that all of the form-data should not be validated when submitted.

Example Code Output
A form with two submit buttons (with and without validation):
<form action="save.php" method="POST" novalidate>
     <label for="username">Name:</label>
     <input type="text" id="username" name="username">
     <label for="usermobile">Mobile:</label>
     <input type="text" id="usermobile" name="usermobile">
     <input type="submit" value="Submit">
</form>
